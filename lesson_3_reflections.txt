When would you want to use a remote repository rather than keeping all your work
local?

If you wanted to backup your local code and/or you were collaborating with someone.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

If you weren't ready for your changes to be made public.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

Forks is only for cloning on GitHub. Cloning allows for working on a copy of a codebase. 
Branches allow for working on different versions of a program.

What is the benefit of having a copy of the last known state of the remote
stored locally?

You can start working right away and be assured you are working with the latest copy of the code.
